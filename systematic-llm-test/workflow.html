How you move from raw documents to structured model results

1
Data Preparation
Start with real (or synthetic) original documents. Generate tampered and augmented variants programmatically.
Input

Output
Labelled Image Set
Saved to tampered/ and augmented/
50 : 50 genuine vs tampered target
Design note: Originals in raw/ are never modified. All variants are generated files so you can always regenerate from scratch if the tamper/augment approach changes.
2
Ground Truth Labelling
Every image gets a row in manifest.csv before evaluation runs. This is the source of truth for all analysis.
Fields per image
manifest.csv
image_id · image_path · doc_type
is_tampered · tamper_types · severity
is_augmented · augment_types
image_quality · source_image_id
You do this
Manual / Script Fill
Fill rows as you create variants. The eval loop reads this file — no manifest row = image is skipped.
3
Evaluation Loop
Run evaluate.py. Every combination of image × model × prompt × iteration is called and written immediately to a .jsonl results file.
Nested loop structure
for image in manifest
→
for model in MODELS
→
for prompt_id in library
→
for iteration in 1..N
call Vertex AI API
→
parse response
→
write result row → flush
Rate limit enforced between calls. Retry (×3, exponential backoff) on failure. Crash-safe: results written row by row.

Output: results/eval_<timestamp>.jsonl
One JSON line per result row
4
Analysis
Load the .jsonl into pandas. Join on ground truth fields (already embedded in every row). Compute metrics per slice.
Key questions
Detection performance
Precision / Recall / F1
by model · by prompt strategy
by tamper type · by severity
Consistency
Across iterations
Does the model agree with itself 3/3 times?
Agreement rate per model × prompt.
Failure modes
Where does it fail?
Low severity vs high severity
Digital vs scan-degraded
Stamp vs text tampering
Latency
Cost vs quality
Flash vs Pro latency gap
Is Pro accuracy gain worth 2× latency?
Tip: The structured JSON prompt (v1_structured) gives machine-parseable per-field confidence scores — use these for deeper analysis than a simple yes/no label.
